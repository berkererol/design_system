# Button

Buttons should be used as interactive elements for user-initiated events such as form submissions or confirmation/cancellation on a modal dialog.

## General

This component is an extended & themed version of MUI's button component.

- [Component documentation](https://material-ui.com/components/buttons/)
- [API documentation](https://material-ui.com/api/button/)
- [Source Code](https://github.com/mui-org/material-ui/tree/master/packages/material-ui/src/Button)

## Usage

### Types of button
We support two types of buttons, these 'types of buttons' are referred to as **variants** by MUI.
The _default variant_ is equal to a **Text Button**, whereas `<Buttton variant='contained'>` will render a **Contained Button** which is a button with a background color.

### Colors of button
UX design dictates that the primary action of a user interface (e.g. Submit, OK), should always be highlighted through a _primary color_ `<Button color='primary'>`, whereas secondary actions (e.g. Cancel, Dismiss) should use the default button `<Button>`.

**NOTE:** If you want an anchor tag to resemble the `<Button>` component, any `<Button>` component with an `href` prop will automatically render the approriate tag.

export const buttonSource = `
// Render a contained button with primary color
  <Button variant="contained" color="primary">
      Contained-primary Button
  </Button>
// ***
// ***
// Render an anchor tag that resembles the same Button component
  <Button href="https://www.onespan.com" variant="contained" color="primary">
      Contained-primary Button
  </Button>
`;
